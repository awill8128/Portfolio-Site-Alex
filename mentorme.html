Background

Problem

How might I… showcase my skills and processes as a UX Designer and information about myself to recruiters and hiring managers?

Outcome

I coded a website to use as my portfolio.

Challenges

See Takeaways below for more details.

Research

By searching for UX Design job listings on several platforms (Linkedin, Indeed, BuiltIn to name a few), I learned it is common and expected for UX Designers to have a portfolio of their case studies. Having just finished a course in frontend development, I decided to put my skills to work by coding my own website.

Before starting to code, I looked to other UX Designers’ portfolios for inspiration. Thinking about the website from my goal audience’s perspective (recruiters and hiring managers), I wanted my portfolio to be…

  • Approachable and intriguing
  • Minimalistic in design and layout to reflect my style
  • Responsive to ensure the portfolio can be used in many contexts
  • Accessible to demonstrate my commitment to accessibility

Creating the Website

With some wireframes and all the content on hand, I began coding using Atom.

I wrote the structure and content of the website with HTML. I added some styling using CSS. Then, using Javascript, I added interactivity (hamburger menu for navigation on smaller screen sizes).

Throughout this process, I used a repository on GitHub to commit changes and trial changes using branches.

Testing & Iterations

With a working website through GitHub, I tested my portfolio with four users. Here are the changes that were made based on test results.

Color Scheme

During testing, participants were asked to give overall impressions of the website. Many pointed to the consistent use of a color scheme when talking about the website looking professional.

Furthermore, accessibility testing on color contrast showed hover/focus states on my buttons did not meet WCAG 2.0 requirements to be AA compliant.

I returned to my goals of making my portfolio "approachable" and "intriguing". Upon analysis of participant remarks regarding what made the porfolio professional, I realized the answer was the consistent use of a color scheme for a cohesive look rather than the use of "professional colors".

I adjusted the color scheme to use brighter colors to help me meet my goals as well as keep the website looking professional. I also fixed color contrast issues for focused & hover states on site elements to be AA compliant.

"It looks very professional... I love the consistency." - Test Participant

Solution

  • Adjust color scheme to incorporate more approachable and intriguing colors
  • Fix all text/background colors to be AA compliant

Figma Embedded Prototype

Based on participant feedback (see caption quote on the left), I decided to move the embedded Figma prototype to the top of the case study before diving into the design process.

But I ran into the challenge of slow loading times (the prototype was previously at the bottom of the case study allowing for the media file to load while users browsed the content above it). Additionally, the prototype was too small to be interactive on mobile screens.

I created a responsive solution that hides the embedded prototype on mobile screens and instead shows a button that allows the user to open a new window to view the prototype.

"In the world of woodworking reddit, you always put the final product at the top before going into how it was built." - Test Participant

Solution

  • Hide the embedded prototype on mobile screen sizes
  • Add button with link to Figma prototype that opens in a new window

Case Study Navigation

While using the navigation links and back to top button on a case study, one participant noted the back to top button does not lead them back to the navigation and the navigation disappears once the user scrolls past it..

Two participants who are full-time recruiters appreciated the clear top navigation bar that helped them find everything they needed quickly.

I designed a new navigation on case study pages that would be just as easy to use as the top navigation bar.

"On a page this long, you definitely need navigation." - Test Participant

Solution

  • Add sticky left column with navigation links
  • Add button with link to Figma prototype that opens in a new window

What's next?

While lots of hard work and time went into this website, there is always room for improvement. Next steps in improving my portfolio would include...

  • Improving accessibility by programming focus order on all pages
  • Continuing user testing by asking hiring managers and recruiters for feedback
  • Measuring the impact of this project by landing a job in UX Design ;)

Takeaways

Appreciation for developers

I took a frontend development course and decided to code my own website to acquire foundational understanding of HTML and CSS. I knew this would help me understand the design constraints and help collaboration with dev teams. What I didn't expect was how much of an appreciation I would develop (pun intended) for coders and programmers. I truly loved learning something new and problem-solving aspects of coding but I'm happy to hand off this job to someone more detail-oriented and patient than me.

Accepting my own limitations

I found accessibility to be much more difficult than expected. Coming from a background of working with people with disabilities, I wanted to demonstrate my commitment to accessibility through my own portfolio. Beyond color contrast checks and including alternative texts for all visual elements, I quickly realized my limitations on addressing accessibility issues. Thankfully there are so many tools for website building and hosting these days that have much bigger capabilities for making my portfolio accessible. That is one of the reasons why you may be reading this case study from a website I did not code myself.
